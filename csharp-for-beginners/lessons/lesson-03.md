# Урок 3: Променливи и типове данни

## Какво ще научиш

До края на този урок ще можеш да:

- обясниш какво е променлива с човешки думи;
- създадеш променлива за текст, цяло число и дробно число;
- разпознаваш трите важни части: тип, име и стойност;
- показваш стойността на променлива в конзолата;
- променяш стойността на вече създадена променлива;
- избираш подходящ тип за прости данни.

## Защо това е важно

В Урок 2 програмите ни показваха готов текст.

Това е полезно, но истинските програми трябва да помнят информация:

- име на играч;
- точки в игра;
- температура;
- цена;
- съобщение.

Променливите са първата стъпка към програма, която не е просто плакат с текст, а малка машина,
която пази и използва данни.

## История с Byte

Byte подрежда малки кутии на бюрото си.

На първата кутия пише `name`, а вътре има бележка:

```text
"Мая"
```

На втората кутия пише `score`, а вътре има число:

```text
15
```

Byte е доволен:

> Byte: "Когато кутията има име, мога да намеря стойността по-късно. Но трябва да знам и какъв вид стойност има вътре."

Точно това правят променливите. Те са именувани места за пазене на стойности.

## Основно обяснение

Променлива е именувано място за пазене на стойност.

В C# една проста променлива изглежда така:

```csharp
string name = "Мая";
```

Нека я прочетем бавно:

- `string` е типът данни. Той казва, че стойността е текст.
- `name` е името на променливата.
- `=` означава "сложи тази стойност в тази променлива".
- `"Мая"` е стойността.
- `;` завършва инструкцията.

Много начинаещи мислят, че `=` означава "равно е" като в математиката. В C# тук е по-полезно да го
четем така:

```text
сложи стойността отдясно в променливата отляво
```

### Първите типове данни

Типът данни казва какъв вид стойност пазим.

В този урок ще използваме три типа:

| Тип | За какво служи | Пример |
|---|---|---|
| `string` | текст | `"Иван"` |
| `int` | цяло число | `15` |
| `double` | дробно число | `22.5` |

Текстът се пише в кавички.

Числата не се пишат в кавички.

```csharp
string city = "София";
int age = 15;
double temperature = 22.5;
```

## Визуално обяснение

Можеш да мислиш за променливите като за кутии с етикети:

```text
+----------------+
| name           |
| "Мая"          |
+----------------+

+----------------+
| score          |
| 15             |
+----------------+

+----------------+
| temperature    |
| 22.5           |
+----------------+
```

Името е етикетът на кутията.

Стойността е това, което е вътре.

Типът казва какъв вид нещо очакваме вътре:

```text
[ тип ] [ име ] = [ стойност ];

string  name    = "Мая";
int     score   = 15;
double  height  = 1.72;
```

## Първи пример

Ето малка програма, която пази информация за Byte:

```csharp
string robotName = "Byte";
int batteryLevel = 87;
double temperature = 22.5;

Console.WriteLine("Име:");
Console.WriteLine(robotName);

Console.WriteLine("Батерия:");
Console.WriteLine(batteryLevel);

Console.WriteLine("Температура:");
Console.WriteLine(temperature);
```

Резултат:

```text
Име:
Byte
Батерия:
87
Температура:
22.5
```

Забележи разликата:

```csharp
Console.WriteLine(robotName);
```

Това показва стойността на променливата.

```csharp
Console.WriteLine("robotName");
```

Това показва самия текст `robotName`, защото е в кавички.

## Разбивка на кода

Нека разгледаме този ред:

```csharp
int batteryLevel = 87;
```

Той казва:

```text
Създай променлива за цяло число.
Името й е batteryLevel.
Сложи вътре стойността 87.
```

После:

```csharp
Console.WriteLine(batteryLevel);
```

казва:

```text
Намери променливата batteryLevel.
Вземи стойността вътре.
Покажи я в конзолата.
```

### Промяна на стойност

Променлива се казва "променлива", защото стойността й може да се промени.

```csharp
int points = 10;
Console.WriteLine(points);

points = 15;
Console.WriteLine(points);
```

Резултат:

```text
10
15
```

Първия път създаваме променливата:

```csharp
int points = 10;
```

Втория път само сменяме стойността:

```csharp
points = 15;
```

Не пишем `int` отново, защото променливата вече съществува.

## Чести грешки

### Грешка 1: Текст без кавички

```csharp
string name = Byte;
```

Защо е проблем:

C# ще търси променлива с име `Byte`. Ако искаме текст, трябва да използваме кавички.

Поправка:

```csharp
string name = "Byte";
```

### Грешка 2: Число като текст

```csharp
int age = "15";
```

Защо е проблем:

`int` очаква цяло число, а `"15"` е текст, защото е в кавички.

Поправка:

```csharp
int age = 15;
```

### Грешка 3: Дробно число с грешен тип

```csharp
int temperature = 22.5;
```

Защо е проблем:

`int` пази цели числа. `22.5` има дробна част.

Поправка:

```csharp
double temperature = 22.5;
```

### Грешка 4: Използване на променлива преди да е създадена

```csharp
Console.WriteLine(score);
int score = 20;
```

Защо е проблем:

Компютърът чете отгоре надолу. Когато стигне до `score`, такава променлива още не съществува.

Поправка:

```csharp
int score = 20;
Console.WriteLine(score);
```

### Грешка 5: Показване на името вместо стойността

```csharp
int score = 20;
Console.WriteLine("score");
```

Резултатът ще бъде:

```text
score
```

Поправка:

```csharp
int score = 20;
Console.WriteLine(score);
```

## Как мисли програмистът

Когато ти трябва променлива, не започвай с писане веднага.

Първо си задай въпроси:

- Каква информация искам да пазя?
- Текст ли е, цяло число ли е, или дробно число?
- Какво име ще бъде ясно след 5 минути?
- Трябва ли стойността да се показва?
- Ще се промени ли стойността по-късно?

Пример:

```text
Искам да пазя точки в игра.
Точките са цяло число.
Името score е кратко и ясно.
```

Код:

```csharp
int score = 0;
```

Byte би казал:

> Byte: "Етикетът е ясен. Типът е ясен. Стойността е ясна. Мога да работя."

## Мини проект

### Карта на герой

Направи малка конзолна карта на герой.

Програмата трябва да пази в променливи:

- име на героя;
- ниво;
- точки;
- любимо умение;
- височина или друга дробна стойност.

Примерен резултат:

```text
+----------------------+
| Карта на герой       |
+----------------------+
Име:
Алекс
Ниво:
3
Точки:
120
Умение:
бързо мислене
Височина:
1.68
```

Преди да пишеш код, планирай:

```text
1. Кои стойности ми трябват?
2. Какъв тип има всяка стойност?
3. Как ще се казва всяка променлива?
4. В какъв ред искам да покажа информацията?
```

## Тест за самопроверка

1. Какво е променлива?
2. Какво е стойност?
3. Какво казва типът данни?
4. Кой тип би избрал за `"Мария"`?
5. Кой тип би избрал за `42`?
6. Кой тип би избрал за `3.14`?
7. Каква е разликата между `Console.WriteLine(score);` и `Console.WriteLine("score");`?
8. Защо не пишем типа отново, когато само сменяме стойността на променлива?

## Обобщение

В този урок научи:

- променливата е именувано място за пазене на стойност;
- `string` пази текст;
- `int` пази цяло число;
- `double` пази дробно число;
- текстът се пише в кавички, а числата не;
- `Console.WriteLine(name);` показва стойността на променливата;
- `Console.WriteLine("name");` показва текста в кавичките;
- стойността на променлива може да се промени.

Най-важната идея:

```text
Типът казва какво може да има в кутията, името казва как да я намерим,
а стойността е това, което пазим вътре.
```
